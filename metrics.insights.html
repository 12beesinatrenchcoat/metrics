    Closed #1095 refactor(metrics/insights): progressive display in lowlighter/metrics

    Split the plugins computation so the page can load in "chunks" rather than "all at once"

    Intended result is that a GitHub user insights will start being displayed even if not completly loaded, and each "pending plugin" will be displayed as "in progress" and will display output once plugin is completed.

    This way we can be faster and display quick plugins results for end users, while still running more time consuming plugins in the background
